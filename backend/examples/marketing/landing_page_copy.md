Section-by-section copy skeleton for a high-converting landing page. Replace `{{...}}`.
Order matters: outcome → proof → how → objections → action.

## 1. Hero
Headline: {{specific_outcome}} without {{main_pain}}
  (Formula: [desirable outcome] + [removed pain/effort]. Concrete > clever.)
Subhead: {{product}} helps {{ICP}} {{do_the_job}} in {{timeframe}}.
Primary CTA: {{action_verb}} {{noun}}  (e.g. "Start free", "Book a demo")
Secondary CTA: "See how it works"
Trust strip: logos / "{{N}}+ {{ICP}} use {{product}}" / rating.

## 2. Problem (agitate briefly)
- 3 bullets naming the status-quo pain in the customer's words.
- One line: "There's a better way."

## 3. Solution / How it works (3 steps)
Step 1: {{action}} → Step 2: {{action}} → Step 3: {{result}}
Each step: 1 verb headline + 1 sentence + optional screenshot.

## 4. Features → Benefits (3–6)
For each: **{{Feature}}** — so you can {{benefit}}.
(Always translate feature into the outcome it unlocks.)

## 5. Social proof
- 2–3 testimonials, each with a name, role, and a NUMBER (saved X hrs, +Y% Z).
- Customer logos.

## 6. Objection handling / FAQ (4–6)
Pricing, security, migration, "is this for me?", cancellation, support.

## 7. Final CTA
Restate the outcome + remove risk ("no card required", "cancel anytime").
Headline: Ready to {{outcome}}?  Button: {{primary_CTA}}

Rules:
- One primary CTA repeated; don't dilute with competing actions.
- Specific numbers beat adjectives. Cut "powerful/seamless/robust".
- Read the hero alone — a stranger should grasp what it is + who it's for in 5s.
