1. AI & Model Stack
Astra's core architectural decision: self-hosted open-source models on GCP for the heavy lifting, with Gemini Flash as the cross-agent orchestration layer and the one required GCP touchpoint. This gives cost control at scale, no API rate limits, and a moat competitors cannot easily replicate.

Why self-hosted models, not OpenAI or Anthropic APIs?
* Cost at scale — API pricing is fine for prototypes. At 1,000 active founders running 6 agents each, the API bill becomes the business. Self-hosted inference cost drops with volume.
* No rate limits — agents run 24/7. API rate limits interrupt autonomous workflows at exactly the wrong moment. Self-hosted has no ceiling.
* Competitive moat — a competitor can replicate a GPT-4 wrapper in a weekend. They cannot replicate a self-hosted, GCP-optimized, fine-tuned model stack without starting from scratch.

The Approval Queue — staying in control without being annoying
Agents do not ask for approval on every action — only the ones that matter. The system learns which actions the founder consistently approves and gradually reduces interruptions.

4. Founder Dashboard Spec
The dashboard is where the founder lives. Every agent reports through it. Every output flows through the approval queue. The founder never needs to open another tool.


Panel
	What Founder Sees
	What Founder Can Do
	Agent Status Ring
	Visual ring per agent: Running / Waiting for Approval / Completed / Idle. Updated in real time.
	Click any ring to open the agent's full activity log.
	Approval Queue
	Prioritized inbox of every decision needing human input. Cards show: what the agent did, what it recommends, why it is asking.
	Approve (1 click), reject (with reason), or edit the output before approving. Agent resumes immediately.
	Agent Chat Interface
	Dedicated chat window per agent. The founder talks to any agent directly at any time.
	Natural language: 'Legal Agent — draft an NDA for a contractor.' Agent responds with full company memory context.
	Company Timeline
	Chronological feed of every action every agent has taken — filings, launches, emails sent, features built.
	Filter by agent, date, or task type. Download any output (doc, report, code) directly.
	Credit Balance & Usage
	Current balance, monthly usage by agent, projection of how long credits last at current burn rate.
	Purchase credits inline. Upgrade plan. Set per-agent credit limits.
	Weekly Digest Preview
	Ops Agent summary: what happened, what worked, what needs attention, recommended priorities for coming week.
	Approve and send to co-founders, or ask Ops Agent to revise before it goes out.

6. Risk Register
Honest risks. Real mitigations. These are the technical risks that can kill the build — read them once and assign owners.


Risk
	Level
	Mitigation
	Self-hosted model quality inconsistent
	HIGH
	Start with Llama 3.1 70B — well-benchmarked. Run evals on all agent task types in Week 1 before building on top. Fall back to larger model if quality is insufficient.
	GCP GPU hosting costs spike
	HIGH
	Hard per-founder compute budgets enforced at infrastructure layer. Batch non-urgent agent tasks. Use 4-bit quantized models where quality allows. Monitor burn rate weekly vs. per-customer COGS target.
	Stripe Atlas Computer Use breaks
	HIGH
	Build direct Delaware registered agent API integration as fallback (Harvard Business Services, InCorp). Computer Use is primary — API is insurance. Do not block launch on Computer Use perfection.
	Legal liability from document generation
	HIGH
	Explicit disclaimer on all documents: 'AI-generated document preparation — not legal advice. Review with a licensed attorney before signing.' Never use the phrase 'legal advice' anywhere in the product. Get disclaimer reviewed by one attorney before launch.
	Mercury bank account not feasible in time
	MED
	Flag as stretch goal from Day 1. Test feasibility in Week 7–8. If UI automation is too fragile, remove from pitch entirely. Entity formation works without it.
	Too broad — agents shallow across 6 functions
	HIGH
	Ship 3 great agents before 6 mediocre ones. Phase 1: Legal + Research + Web. Phase 2: Technical + Marketing. Ops Agent last. Depth beats breadth. Do not build all 6 in parallel and ship nothing good.
	Founders churn after Launch fee — never subscribe
	MED
	Design Launch package to leave clear gaps that Build fills. LLC is done but marketing, coding, and ops agents only activate on Build plan. Setup creates appetite — subscription delivers.
